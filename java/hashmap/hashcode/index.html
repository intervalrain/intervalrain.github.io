[Java] HashMap中的hashCode設計原理

why hashcode >>> 16
April 22, 2022 · 1 分鐘 · Rain Hu | Suggest Changes
目錄
  • 程式碼
  • h » 16 的用途
  • key.hashCode() ^ (h » 16)

程式碼#

static final int hash(Object key){
    int h;
    return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
}

h » 16 的用途#

  • h是key.hashCode()，h >>> 16代表的是取其高位的16位

key.hashCode() ^ (h » 16)#

  • 這與 Java1.8 中 tab[(n-1) & hash] 的原理有關
    static int indexFor(int h, int length){
    return h & (length - 1);
}
    • 返回的值即為陣列的下標。
    • 大多數情況下，capacity 都小於2^16，所以在此的 & 運算，只會對 h 的低16位進行 & 運算。
    • 若將高位16位也加入計算，可以增加下標的發散度，避免衝突的次數。
    • 而使用 XOR 的原因是，更較於 AND 或 OR 均勻，因為 AND 會使結果趨向於 0，OR 會使結果趨向於 1。
